• 예약어: 이미 쓰임이 정해져있는 키워드 (let 등)

변수

변수 선언하기

변수: 특정한 값을 저장할 수 있다.

let으로 시작하는 명령을 선언문이라고 한다.

대입 연산자(=)보다 더하기 연산자의 순위가 높다.

변수를 선언함과 동시에 값을 대입하는 행위를 초기화(initialization)라고 한다.

변수 선언은 항상 결괏값이 undefinde다.

console은 브라우저에서 기본제공해준다. 따라서 undefinde가 안 나옴(.log 아님)

변수 이름짓기

한국어, 이모티콘, 한자 등 모두 가능하며 특수문자 중에서는 $, _(언더스코어)만 가능하다.

첫글자로 숫자는 안 된다.(css class도 똑같음)

CamelCase

let, class, const 등은 변수명으로 불가능하다.

let은 예약어가 아니다.

let의 예약어 오류가 다른 이유: es6가 나오기 전에 let은 완벽히 변수가 될 수 있었다.
그래서 let만이 유일하게 예약어가 아니었던 적이 있어서 에러메시지가 다르다고 한다.
예를 들어) var let = 3의 경우에는 올바른 문법이다.

변수 수정하기

수정할 때는 let을 붙이면 안 된다. 붙일 경우 오류가 뜬다.
메세지: Identifier '중복변수명' has already been declared

수정했을 때 undefinde가 안 나오는 이유: let의 역할, let이 없을 때는 코드가 식이라서 대입한 값이 결괏값으로 출력되지만 let이 앞에 붙는 순간 선언문이 된다.
문(statement)은 식과 다르게 결괏값이 없고 식의 자리에 사용할 수 없다.

선언할 때는 undefinde 바뀔 때는 결괏값

변수에 넣은 값을 비우기: 1. undefinde를 대입 2. null을 대입

변수 활용하기

let string2 = string(이미 선언한 변수임)
자신을 대입할 수도 있다.
let number = 5;
number = number + 3
number의 최종값: 8

number = number + 3;은 이와 같다. number += 3;

변수는 계산된 값을 저장할 때도 사용하지만, 중복을 줄이려고 사용하기도 한다.

상수(const)와 var

상수

한 번 선언한 것은 바꿀 수 없다. 에러: ssignment to constant variable.

상수는 수정이 불가하기 때문에 꼭 값을 넣어줘야 한다. (const aaa;으로 끝나며 안 됨.)

var

var는 변수이며, es5까지 사용되던 키워드이다.

var는 재선언이 가능하다.

나사가 하나 빠진 느낌...? es6에서 let으로 단점이 보완됐다. 하지만 전에는 var를 써왔기 때문에 알아둬야 한다.
