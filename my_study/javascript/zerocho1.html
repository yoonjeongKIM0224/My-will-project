불 값(boolean, 값들의 비교)

불 값(boolean, 값들의 비교)

boolean 값은 ;로 감싸면 안 된다.

5>3, 3>6 등을 콘솔창에 치면 불 값이 나온다.

== (같다는 =을 두 번, = 한 번은 대입)

NaN == NaN은 false

NaN != NaN은 true

true > false 는 true (이유: true는 1이라는 의미와 비슷함)

문자열도 비교가 가능하다.
ex)'b' > 'a' 사전 순으로 뒤에 나올 수록 크다.

특수문자도 비교가 가능하다.
ex) &은 65286이다. '&'.charCodeAt() *왜 다르징 ㅎ

자료형이 숫자가 아닌 자료형은 숫자로 변환 후 비교한다.
'3'은 변환 가능하지만, 'abc' 등은 숫자로 변환이 불가하여 NaN으로 인식된다.
(NaN은 !=NaN을 제외하고는 무조건 false가 나온다.)

== (같다.) === (자료형까지도 같다.)

1 == true (값: true)
1 === true (값: false)

1 != '1' (값: false) 1 !== '1' (값: true)

5 + 4 * 3 === 27 (문제! 동영상 13분 경)

논리연산자

논리연산자

! 연산자: 참을 거짓으로, 거짓을 참으로 만들어 준다.
ex) !true(false), !flase(true)

다른 자료형을 불 값으로 변환할 수 있으며, 2가지 방법이 있다.
ex)'a' 는 불값으로 변환하려는 불값이 아닌 자료형
1. boolean('a');
2. !!'a' (기존값의 반대 -> 기존값의 반대의 반대(처음 값))

false, ''(빈문자열), 0, NaN, undefinde, null, document.all을 제외하고는 거의 true이다. *위의 두 가지 방법으로 확인 가능

!(2 < 0) 값: true
!2 < 0 값: flase (우선순위표 확인)

undefined와 null(빈 값들)

undefined

console.log 함수는 return 값이 존재하지 않는 함수이기에 console에서 실행하게 된다면 undefined가 반환
console.log에서 undefined가 나오는 부분 앞에는 작게 화살표 ( < )가 붙어있다.

undefined의 불값은 false다. 단, undefinde와 false가 같지는 않다.

undefined는 기본값이라는 의미가 있다.

null

다른 자료형으로는 null이 있다. 단, undefined처럼 빈 값을 의미하지만 같지는 않다.

의도적으로 빈 값을 나타낼 때 사용한다.

undefined와 불값은 false로 같지만, 자료형은 다르다.

null의 자료형은 null이지만, typeof를 돌려봤을 때 object가 나오는 이유는 자바스크립트에서 유명한 버그이다.
언어가 만들어진 초창기 실수 때문에 object가 됐다. 따라서 null인지 확인하기 위해서는 === null을 사용해야 한다.

null은 역할이 없다.
